# Option Pricer
	A unifying high-performance (C++) option pricing tool

### How to run:
1. double-click **pxOptionGui.exe**;
2. update GUI inputs, if necessary. See GUI tooltips & [more info below](#option) for business logic;
3. click the **Compute** button.
	
Dir Structure (automatically generated):
- qlcpp: sub-dir containing the C-dll file and **option.json**, which is used to set up the GUI
- qlinput: sub-dir for storing the [parameters file](#params), which is generated by the GUI and needed by C-dll
- qllog: sub-dir for storing pricing result files (dpx\_\*.txt and err\_\*.log)


System requirements:
- Windows OS

No installation needed! Download and enjoy it for free.\
*If you have questions on the Option pricer*, [Contact me](mailto:ql12@cornell.edu).

### Pricing method supported (VALUE for KEY[deriv_px_method_id]):
<a name="option"></a>
- antithetic_mc: Anthithetic Monte Carlo
- antithetic_ls_mc: Anthithetic least-square Monte Carlo
- binom_tree: Binomial-tree with Trigeorgis-Params (minimum steps: 365)
- bsm_formula: (BDA) Black-Sholes-Merton formula
- divamount_bda: BDA recombining binomial tree
- divamount_bt: Non-Recombining binomial tree
- finite_difference: Finite difference. See KEY[diffuse_method_id], with VALUE:
  - crank_nicolson: Crank-Nicolson
  - explicit_finite: Explicit difference
  - implicit_finite: Implicit difference

### Underlying supported (VALUE for KEY[underlying_id]):
- common_stock: Stock (with fixed_dividend, div_frequency, next_div_date)
- stock_index: Stock index (with continuous_div_yield)
- foreign_currency: Foreign currency (with foreign_risk_free)
- futures

#### Cash (dollar) dividends
ONE dividend:
- divamount_bt
- divamount_bda

Multiple dividends:
- bsm_formula
- finite_difference

### Notes
<a name="params"></a>
The parameters file with pair entries [KEY VALUE] is utilized by C-dll to compute option price, delta, gamma, and implied volatility. Extra entries that are not used by a chosen method are ignored by C-dll.

The C-dll expires **two years** after its compilation, and will be replaced by a **new version**.
